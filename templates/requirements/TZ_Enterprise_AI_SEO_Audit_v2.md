enterprise_md_content = """# ТЗ v2.0: Enterprise-сервис комплексного AI- и SEO-аудита на базе Sitemap.xml

## 1. Цель проекта
Создание системы автоматизированного аудита полного цикла, которая последовательно обходит страницы сайта (начиная с `sitemap.xml`) и проводит глубокий технический, семантический и "генеративный" (GEO) анализ каждой единицы контента.

## 2. Архитектура и сетевой уровень

### 2.1. Метод обхода и рендеринг
1. **Headless Chrome Integration:** Использование Puppeteer или Playwright для полной эмуляции браузера.
2. **JavaScript Rendering SEO:** Сравнение исходного кода (Server-Side) и финального DOM (Client-Side) для поиска скрытого контента.
3. **Crawl Budget Management:**
   - Поиск бесконечных редиректов.
   - Фильтрация мусорных URL с параметрами.
   - Замер **TTFB** (Time to First Byte).

### 2.2. Core Web Vitals (CWV)
Для каждой страницы сервис должен фиксировать:
- **INP (Interaction to Next Paint):** Оценка интерактивности (стандарт 2024 года).
- **LCP (Largest Contentful Paint):** Скорость загрузки основного контента.
- **CLS (Cumulative Layout Shift):** Визуальная стабильность.

## 3. Модуль GEO (Generative Engine Optimization)

### 3.1. Анализ для RAG-систем (Gemini 2.5)
Интеграция с API Gemini 2.5 для оценки "понятности" контента для LLM (Perplexity, SearchGPT):
1. **Entity Extraction:** Извлечение именованных сущностей (NLP Entities) и оценка их тематической связности.
2. **Topical Authority:** Оценка глубины раскрытия нишевых вопросов.
3. **E-E-A-T сигналы:** Автоматический поиск биографий авторов, политик рецензирования и внешних авторитетных ссылок.

### 3.2. Валидация семантики
- Глубокая проверка **JSON-LD** и **schema.org**.
- Контроль схем: `FAQPage`, `Article`, `Organization`, `Product`.

## 4. Глубокая аналитика структуры и индексации

### 4.1. Граф ссылок и перелинковка
1. **Orphan Pages:** Выявление страниц, указанных в `sitemap.xml`, на которые нет внутренних ссылок.
2. **PageRank Sculpting:** Визуализация распределения "веса" между хабами и кластерами.
3. **Международное SEO:** Валидация `hreflang` (проверка обратных ссылок и региональных конфликтов).

### 4.2. Конфликты индексирования
Перекрестный анализ: `robots.txt` vs `meta robots` vs `X-Robots-Tag` vs `Canonical`.

## 5. Семантическая и конкурентная разведка

1. **Keyword Cannibalization:** Поиск страниц, конкурирующих за один и тот же интент.
2. **Content Gap Analysis:** Сравнение семантического ядра сайта с Top-5 конкурентами.
3. **Intent Matching:** Проверка соответствия контента страницы поисковому запросу.

## 6. Безопасность и внешние факторы

1. **Токсичность ссылок:** Интеграция с БД (Ahrefs/Semrush/Majestic) для расчета **Toxic Score** и анализа спамных анкоров.
2. **Security SEO:**
   - Поиск **Mixed Content** (HTTP ресурсы на HTTPS страницах).
   - Сканирование уязвимостей в используемых JS-библиотеках.
   - Проверка корректности и срока действия SSL-сертификатов.

## 7. Требования к реализации в Антигравити

### 7.1. Стек (уточнение)
- **Engine:** Python (FastAPI/Celery для очередей).
- **Browser:** Playwright (Headless).
- **AI:** Google Vertex AI (Gemini 2.5 Flash/Pro).
- **DB:** PostgreSQL (для хранения истории аудитов) + Redis (для очереди URL).

### 7.2. Формат отчета
- Интерактивный Dashboard с сегментацией по критичности (Critical, Warning, Info).
- Экспорт в JSON/CSV для внешних BI-систем.
- Экспорт в PDF для человека-читабельного вида.
